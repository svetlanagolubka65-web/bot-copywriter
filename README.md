# ИИ-помощник по контенту — Telegram-бот

Бот для создания текстов в социальные сети. Задаёт уточняющие вопросы и генерирует готовый контент через Groq API (Llama 3.3 70B).

**Бот в Telegram:** [@my_ai_svethelp_bot](https://t.me/my_ai_svethelp_bot)

**Доступ:** приватный — отвечает только владельцу (Telegram ID из `ADMIN_IDS` в `.env`), остальным пользователям приходит сообщение «бот приватный».

## Что умеет

| Кнопка | Что создаёт |
|--------|-------------|
| 📝 Написать пост | Пост для Telegram/Instagram (тема + аудитория + тон) |
| 📖 Описание курса | Продающий текст для онлайн-курса |
| 🎬 Сторис-сценарий | Сценарий из 4–5 слайдов |
| 🎠 Карусель | Сценарий карусели на 7 слайдов (крючок → суть → CTA) |
| 📊 Моя история | Личная история для соцсетей |

Дополнительно:
- Бот приватный (см. «Доступ» выше), поэтому пишет всегда от женского лица без уточнений при знакомстве
- `/settov` — загрузи примеры своих постов, бот запомнит твой стиль
- Кнопки после генерации: ✏️ Доработать / 🔄 Новый вариант / #️⃣ Хэштеги / 🏠 Главное меню

## Бот-консультант по услугам (intake_bot.py)

Второй, отдельный Telegram-бот. Плашки услуг в разделе «Обо мне» на лендинге (`index.html`) ведут не сразу в личку Светланы, а в этого бота (`t.me/<бот>?start=<услуга>`):
- бот показывает карточку услуги (что входит, цена, срок);
- отвечает на свободные вопросы через Groq, опираясь только на реальные цены и FAQ с лендинга (если не уверен — сам пишет, что уточнит у Светланы, и пересылает вопрос ей);
- либо собирает короткий бриф (задача / сроки / бюджет) и сразу отправляет его Светлане, а клиенту отвечает, что она свяжется с ним;
- для услуги **🎨 AI-визуал** сначала спрашивается подтип — 📸 Фото / 💌 Открытка / 🎬 Ролик:
  - **Фото** → отдельный флоу заказа AI-фото: клиент выбирает пресетный стиль (пляж, вечернее платье, деловой образ, студийный портрет, повседневный) или описывает свой, затем присылает фото. В чат уведомлений уходит и фото, и уже готовый текст промпта для Gemini (см. [`prompts/photo_styles.md`](prompts/photo_styles.md)) — Светлане остаётся скопировать его и прислать результат клиенту лично; оплата пока не встроена.
  - **Открытка** / **Ролик** → обычный бриф (задача/сроки/бюджет), как у остальных услуг, с пометкой типа в уведомлении.

Подробности флоу — в [`ARCHITECTURE.md`](ARCHITECTURE.md#архитектура-бот-консультант-по-услугам-intake_botpy). Запуск — `python intake_bot.py`, токен — `INTAKE_BOT_TOKEN` в `.env`.

**Бот в Telegram:** [@Superkosultantbot](https://t.me/Superkosultantbot). Ссылки на 7 плашках в `index.html` уже ведут на него.

## Бот-помощник курса «Светлячок» (svetlyachok_bot.py)

Третий, отдельный Telegram-бот — личный помощник для участников курса «НейроСвета» (аудитория 50+). В отличие от `intake_bot.py` (лиды с лендинга) и `bot.py` (генерация контента для самой Светланы), этот бот ведёт ученика по задаче через главное меню из 8 кнопок, а не заставляет искать нужный раздел самому:

| Кнопка | Что делает |
|--------|-------------|
| 🖼 Создать картинку | Готовые промпты для Kandinsky / Шедеврум / Midjourney |
| 🎥 Сделать видео | Готовые промпты для оживления фото (Kling, Pika) |
| 🎵 Написать песню | Готовые заготовки под Suno |
| 💬 Освоить ChatGPT | Короткий мини-урок прямо в чате |
| 📚 Найти урок | Ссылка на уроки курса (`LESSONS_URL`) |
| 💡 Получить промпт | Случайный промпт из всех разделов, с кнопкой «Ещё» |
| 🤖 Выбрать нейросеть | Короткая шпаргалка — какой сервис под какую задачу |
| 👩 Позвать Светлану | Ссылка на запись к Светлане (`CONSULT_LINK_URL`), Светлане приходит уведомление |

**Доступ** — только по общему коду курса (`ACCESS_CODE` в `.env`): при первом `/start` бот просит ввести код, дальше запоминает пользователя в `svetlyachok_access.json` и больше не спрашивает. Если `ACCESS_CODE` не задан — бот открыт всем (как и `ADMIN_IDS` в остальных ботах репозитория).

Промпты в разделах — стартовый пример (по 2 на категорию), не полная библиотека курса. Подробности — в [`ARCHITECTURE.md`](ARCHITECTURE.md#архитектура-бот-помощник-курса-светлячок-svetlyachok_botpy). Запуск — `python svetlyachok_bot.py`, токен — `SVETLYACHOK_BOT_TOKEN` в `.env`.

## Лендинг (index.html)

Статическая страница, без сборки — хостится на GitHub Pages: **[svetlanagolubka65-web.github.io/bot-copywriter](https://svetlanagolubka65-web.github.io/bot-copywriter/)**. Автодеплой: любой `git push origin main` подхватывается сам, отдельного шага деплоя не нужно.

Порядок блоков сверху вниз: шапка → хиро (позиционирование «ИИ после 50», две кнопки — научиться / заказать подарок) → личная история → портфолио (работы показаны сразу как открытые карусели/плееры, без клика — старый лайтбокс убран) → отзывы → услуги → кнопки-мостик → «Почему 50+» → «Что можно сделать сегодня» → процесс → цены → FAQ → цитата → форма заявки → контакты.

Подробности — карусели, структура секций, куда делись старые фото/видео из портфолио, известные ограничения (например, большая base64-картинка логотипа в шапке) — в [`ARCHITECTURE.md`](ARCHITECTURE.md#архитектура-лендинг-indexhtml).

## Как запустить локально

**Требования:** Python 3.10+

```bash
# 1. Клонировать репозиторий
git clone https://github.com/svetlanagolubka65-web/bot-copywriter.git
cd bot-copywriter

# 2. Установить зависимости
pip install -r requirements.txt

# 3. Создать .env из шаблона и заполнить ключи
cp .env.example .env

# 4. Запустить
python bot.py
# или бот-консультант по услугам:
python intake_bot.py
# или бот-помощник курса «Светлячок»:
python svetlyachok_bot.py
```

## Нужные ключи (.env)

| Переменная | Где получить |
|-----------|-------------|
| `BOT_TOKEN` | [@BotFather](https://t.me/BotFather) → /newbot |
| `INTAKE_BOT_TOKEN` | [@BotFather](https://t.me/BotFather) → /newbot (отдельный бот для intake_bot.py) |
| `SVETLYACHOK_BOT_TOKEN` | [@BotFather](https://t.me/BotFather) → /newbot (отдельный бот для svetlyachok_bot.py) |

| `LESSONS_URL` | Ссылка на страницу с уроками курса — используется кнопкой «Найти урок» |

| `ADMIN_IDS` | Твой Telegram ID, через запятую если несколько (узнать через [@userinfobot](https://t.me/userinfobot)) — используется в `intake_bot.py` для уведомлений о заявках и в `svetlyachok_bot.py` для уведомлений о запросах на консультацию |

## Тестирование, код-ревью и security review

**Статус на 2026-07-06: пройдено** (относится к `bot.py` и лендингу). **Статус на 2026-07-11: пройдено** для флоу заказа AI-фото и подтипов услуги «visual» в `intake_bot.py` (см. ниже) — остальной `intake_bot.py` (карточки услуг, Q&A, обычный бриф) отдельного код-ревью/security review всё ещё не проходил, см. Roadmap. Порядок действий для будущих изменений — в [`REVIEW_CHECKLIST.md`](REVIEW_CHECKLIST.md).

- **Тесты** — 57 тестов (`pytest` + Playwright) в [`tests/`](tests/): генерация контента через Groq, многошаговый флоу вопросов, онбординг и профиль, действия с готовым текстом, лендинг (карусели портфолио, переключение видео по вкладкам, ссылки), бот-консультант по услугам (`intake_bot.py`: карточка услуги, свободные вопросы, сбор брифа, уведомления админам, выбор подтипа услуги «visual» и флоу заказа AI-фото, включая защиту от повторного тапа по устаревшей кнопке), бот-помощник курса (`svetlyachok_bot.py`: код доступа, все разделы меню, случайный промпт, вызов Светланы с уведомлением, callback-кнопки). Запуск: `pytest tests/ -v`. Все 57 зелёные.
- **2026-07-23, `svetlyachok_bot.py`** — новый бот для участников курса: главное меню из 8 кнопок (ведёт по задаче, доступ по общему коду курса, хранится в `svetlyachok_access.json`). Пока прошёл только автотесты, отдельный код-ревью/security review — в Roadmap.
- **2026-07-23, `bot.py`** — приватизация бота под одного владельца:
  - Кнопка «Заголовки» заменена на «Карусель» (сценарий из 7 слайдов вместо 7 заголовков).
  - Добавлен `check_owner()` — все хендлеры теперь отвечают только ID из `ADMIN_IDS`, остальным приходит «бот приватный».
  - Убран выбор пола автора при `/start` (кнопки «Женщина»/«Мужчина») — бот всегда пишет от лица владелицы, `gender` зафиксирован как `"f"`.
  - Код-ревью нашло и исправило: `check_owner()` падал с `AttributeError` на update-объектах без атрибута `message` (например, некоторые типы callback-апдейтов) — заменено на `getattr(..., None)`.
  - Тесты синхронизированы с новым поведением (`tests/conftest.py`, `test_2_question_flow.py`, `test_3_onboarding_profile.py`); тест выбора пола удалён как проверяющий убранную функциональность.
- **Код-ревью** нашло и **исправило** 3 проблемы (2026-07-06, `bot.py`):
  - Ответ текстом вместо клика по кнопке (например, на шаге выбора тона) стирал уже собранные ответы (тему, аудиторию) и тихо начинал новый пост — исправлено в `handle_message` (bot.py).
  - Клавиатура выбора пола автора никогда не показывалась пользователю — код существовал, но был недостижим. Теперь `/start` показывает её новым пользователям.
  - Побочно всплывший баг: `send_onboarding()` получал не тот объект (`Message` вместо `Chat`) и упал бы при первом реальном срабатывании исправленного выбора пола — исправлено.
- **Код-ревью нового флоу заказа AI-фото** (2026-07-11, `intake_bot.py`) нашло и **исправило** 1 проблему, отрефакторило дублирование:
  - Кнопка «✅ Готово, отправить» не проверяла, что пользователь всё ещё в режиме `photo_upload`, и не сбрасывала `photo_count`/`photo_style`/`photo_prompt` после завершения заказа. Telegram не отключает клавиатуру на старых сообщениях — повторный тап по уже отработавшей кнопке (например, случайно или на устаревшем сообщении) заново отправлял админам дубль уведомления с промптом. Исправлено: `photo_done` теперь игнорируется вне `mode="photo_upload"`, а `_finish_photo_order` полностью сбрасывает состояние заказа.
  - Три места рассылки по `NOTIFY_CHAT_IDS` (`_notify_admins`, пересылка фото, отправка промпта) дублировали один и тот же `for chat_id: try/except` — вынесено в `_notify_admins`/`_notify_admins_photo`.
- **Security review** — прочитано, изменений не вносилось:
  - Секретов (ключей, токенов, паролей) в коде не найдено (включая новый код флоу AI-фото).
  - `.env` прописан в `.gitignore` (корень и `nejrosveta-agent/`).
  - `.env` никогда не попадал в историю git — проверено по всем коммитам, веткам и «висящим» (unreachable) объектам.
  - Новый флоу не добавляет зависимостей и не парсит пользовательский текст как Markdown (сообщение с промптом намеренно отправляется без `parse_mode`, чтобы свободный текст клиента не сломал разбор и не уронил уведомление «тихо»).
  - `pip-audit` по `requirements.txt`: 1 известная уязвимость — `python-dotenv==1.0.0` (CVE-2026-28684). Код использует только `load_dotenv()`, уязвимый путь (`set_key`/`unset_key`) не задействован — реального риска нет, но обновление рекомендовано (см. Roadmap).

## Roadmap

Задачи и замечания, которые заметили при код-ревью/security review, но осознанно отложили:

| Что | Почему отложено | Приоритет |
|---|---|---|
| Краш в `generate_from_answers`, если в `answers` не хватает полей (например, только `topic` без `audience`/`tone`) — `TypeError` вместо ответа пользователю | В обычном сценарии (вопросы бота по порядку) все поля всегда заполнены, баг проявляется только при нестандартном вызове функции; чинить сразу не стали, чтобы не трогать логику генерации без отдельного тестового покрытия | Средний |
| Мёртвый код в `index.html`: CSS-классы `.f-in`/`.f-btn` и функция `handleSend()` — остались от старой формы заявки, сейчас на странице только кнопка-ссылка в Telegram | Не влияет на пользователей (код нигде не вызывается), удаление — чисто косметическая уборка | Низкий |
| Обновить `python-dotenv` до `1.2.2+` (закрывает CVE-2026-28684) | Уязвимый код-путь у нас не используется, риска нет, но апдейт дешёвый и полезен на будущее | Низкий |
| `GROQ_API_KEY` общий для `bot.py`, `intake_bot.py` и `nejrosveta-agent/` (у ботов свои `BOT_TOKEN`/`INTAKE_BOT_TOKEN`, а вот ключ Groq один) | Требует создания отдельных ключей на console.groq.com и ручной ротации — не блокирует текущую работу | Низкий |
| `users.json` как хранилище пользователей — не переживает пересоздание диска при деплое без volume | Известное ограничение (см. `ARCHITECTURE.md` → «Ограничения»), актуально только при миграции хостинга | Средний |
| `svetlyachok_access.json` (доступ по коду курса) — та же особенность: обнуляется при каждом редеплое на Railway, участникам придётся один раз заново ввести тот же код | Простое решение для MVP без Railway volume; неудобство минимально — код не меняется, а ввести его надо только один раз после каждого обновления бота | Низкий |
| `svetlyachok_bot.py` не проходил код-ревью/security review (только автотесты) | Написан позже основного ревью, требует отдельного прогона по `REVIEW_CHECKLIST.md` перед тем как давать ссылку большому потоку учеников | Средний |
| Промпты в `SECTIONS` (`svetlyachok_bot.py`) — по 2 примера на раздел, а не полная библиотека курса | Осознанный MVP-скоуп; наполнение реальными промптами/уроками — следующий шаг | Средний |
| `intake_bot.py` не проходил отдельный код-ревью/security review (только автотесты) | Написан позже основного ревью 2026-07-06, требует отдельного прогона по `REVIEW_CHECKLIST.md` | Средний |
| `SERVICES`/`GENERAL_FAQ` в `intake_bot.py` дублируют текст цен и FAQ из `index.html` вручную, без общего источника данных | Проще для MVP; при правке цен на лендинге легко забыть обновить бота — см. `ARCHITECTURE.md` → «Ограничения» | Средний |
| Заказ AI-фото (услуга «visual») без приёма оплаты — генерация и доставка результата полностью вручную Светланой | Сначала проверяем спрос на MVP; оплату (Telegram Stars / ЮKassa) добавим отдельно, когда будет понятно, что заказы идут | Низкий |
| Нет лимита на число фото в одном заказе (`photo_upload`) — пользователь может прислать сколько угодно фото подряд, каждое сразу пересылается в чат уведомлений | Обычный клиент присылает 1–3 фото; риск — только случайный/неаккуратный ввод, не злонамеренная атака (в бота нет доступа посторонним, ссылка только с лендинга), спам-фильтров у Telegram Bot API тоже достаточно на практике | Низкий |
| Логотип в шапке `index.html` вшит как base64 (~90 КБ строка), а не отдельный файл — раздувает HTML и мешает читать файл рядом с этим местом | Косметическая уборка, вынести в `assets/` отдельным файлом | Низкий |
| Тексты услуг (`#about`) и старых отзывов на лендинге всё ещё «универсальные», не переписаны под новое позиционирование «ИИ после 50» — обновили только структуру/хиро/новые блоки | Отдельная содержательная задача (переписать копирайтинг), не структурная — осознанно отложили в эту сессию | Средний |
