# Архитектура: ИИ-помощник по контенту (bot.py)

## Стек

| Компонент | Технология |
|-----------|-----------|
| Язык | Python 3.10+ |
| Telegram | python-telegram-bot 21.5 (async) |
| ИИ-модель | Groq API → Llama 3.3 70B Versatile |
| Хранилище пользователей | `users.json` + `users_data.json` (локальные файлы) |
| Конфиг | `.env` через python-dotenv |

## Как работает

```
Пользователь (Telegram)
        │
        ▼
  start() ──── новый юзер? ──▶ gender_keyboard() ──▶ handle_callback(gender_*) ──▶ send_onboarding()
        │
        ▼
  handle_message()
  handle_callback()
        │
        ▼
  ask_next_question()  ←── CONTENT_TYPES[тип]["questions"]
        │
        ▼ (после сбора всех ответов)
  generate_from_answers()
        │
        ▼
  Groq API (Llama 3.3 70B)
        │
        ▼
  Сообщение пользователю + result_keyboard()
```

## Ключевые структуры

### CONTENT_TYPES
Словарь с 5 типами контента. Каждый тип содержит:
- `questions` — список уточняющих вопросов (текст + опционально кнопки)
- `prompt` — шаблон промпта с `{плейсхолдерами}`
- `label` — название для логов

### Поток сообщений

1. Пользователь нажимает кнопку меню → `content_type` сохраняется в `user_data`
2. Бот задаёт вопросы по очереди через `ask_next_question()`
3. Ответ принимается и через кнопку (`qans_*`), и обычным текстом — даже если у вопроса есть кнопки (см. `handle_message`, ветка «Ответ на вопрос в потоке»)
4. Ответы сохраняются в `user_data["answers"]` по ключу вопроса
5. После последнего вопроса → `generate_from_answers()`
6. Промпт собирается через `prompt.format(**answers)`
7. Если пользователь загрузил стиль через `/settov` — он добавляется в системный промпт

### Онбординг и профиль
1. Новый пользователь → `start()` сохраняет его в `users.json` и показывает `gender_keyboard()`
2. Выбор пола (`gender_f` / `gender_m`) → `handle_callback` сохраняет профиль в `users_data.json` (`save_user_profile`) и вызывает `send_onboarding(query.message.chat, name)`
3. Пол хранится в `user_data["gender"]` и влияет на `_gender_note()` — от какого лица генерируется текст

### Callback-кнопки после генерации

| callback_data | Действие |
|--------------|----------|
| `regenerate` | Повторная генерация с теми же ответами |
| `refine` | Запрашивает правки, перегенерирует с учётом |
| `hashtags` | Подбирает 7–10 хэштегов к последнему тексту |
| `main_menu` | Очищает `user_data`, кроме `name`/`gender`/`tov`, возвращает в меню |

### Стиль автора (ToV)
- Команда `/settov` → `user_data["waiting_tov"] = True`
- Следующий текст от пользователя сохраняется в `user_data["tov"]`
- При генерации `tov` добавляется в системный промпт: `"Стиль автора:\n{tov}"`

## Хранение данных

- **`users.json`** — список Telegram ID пользователей (для различения новых и вернувшихся)
- **`users_data.json`** — профиль пользователя: имя и пол
- **`user_data`** — in-memory контекст разговора (сбрасывается при /start или main_menu)

## Ограничения

- `user_data` не персистентный — после перезапуска бота стиль (ToV) сбрасывается
- `users.json` / `users_data.json` хранятся локально — при деплое нужно либо монтировать volume, либо перейти на БД
- `generate_from_answers()` падает с необработанным `TypeError`, если `answers` содержит `topic`, но не хватает остальных полей формата (см. Roadmap в `README.md`) — в штатном флоу вопросов бота не воспроизводится
