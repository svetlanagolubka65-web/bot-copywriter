## Обработка ошибок — Общие правила

### Принципы

1. **Пользователь видит:** простое сообщение на русском, что делать дальше.
2. **Разработчик видит:** технический лог с кодом ошибки, timestamp, контекстом.
3. **Агент не умирает тихо:** при критической ошибке — всегда уведомление.
4. **Retry-стратегия:** Claude — 2 retry с задержкой 10s. Telegram — 3 retry с задержкой 30s. RSS — 1 retry, потом пропуск источника.

### Матрица сообщений пользователю

| Категория | Триггер | Сообщение Светлане |
|-----------|---------|-------------------|
| ИИ недоступен | Claude API error 5xx | «ИИ сейчас перегружен. Попробую снова через 10 минут — если не придут посты, напиши /retry» |
| ИИ — неверный ключ | Claude API 401 | «Ошибка доступа к ИИ. Требуется обновить настройки — сообщите разработчику.» |
| ИИ — лимит исчерпан | Claude API 429 | «Достигнут дневной лимит запросов к ИИ. Посты придут завтра утром.» |
| Нет новостей | 0 статей после фильтра | «Сегодня крупных ИИ-новостей не нашла. Хочешь задать тему сама? Напиши /idea [тема]» |
| Память недоступна | Sheets API error | «Сохранение в архив временно не работает. Посты сгенерированы — выбирай как обычно.» |
| Общий сбой | Любая uncaught exception | «Что-то пошло не так. Попробуй /retry. Если ошибка повторится — напиши разработчику.» |

### Лог-формат (для n8n Code нод)

```javascript
// Уровни: INFO, WARN, ERROR, CRITICAL
const log = (level, agent, message, context = {}) => {
  console.log(JSON.stringify({
    level,
    agent,       // 'agent1' | 'agent2' | 'agent3'
    message,
    timestamp: new Date().toISOString(),
    ...context
  }));
};

// Пример использования:
log('ERROR', 'agent2', 'Claude API failed', { 
  status: 503, 
  retry: 1, 
  format: 'format1_news' 
});
```
