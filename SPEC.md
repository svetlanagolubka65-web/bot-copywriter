## 1. Контекст и боль

Светлана — эксперт по ИИ, владелец онлайн-школы, автор Telegram-канала НейроСвета. Каждый день нужно публиковать контент, чтобы удерживать аудиторию (новички в ИИ, предприниматели, разработчики) и конвертировать её в покупателей курсов.

**Три точки страдания:**

1. **Придумать тему** — каждый день искать что написать, следить за новостями ИИ
2. **Найти новость** — мониторить каналы конкурентов, RSS, X/Twitter вручную
3. **Написать текст** — адаптировать в своём стиле, не безликий ИИ-текст

**Цель агента — закрыть все три точки:**

- Экономия 2–3 часов в день
- Регулярность 7/7 дней без пропусков
- Посты в голосе Светланы, а не шаблонный ИИ
- Мягкая продажа одного из 2 курсов в нужный момент

> ⚠️ **Ключевое ограничение:** Светлана ВСЕГДА видит пост перед публикацией и нажимает кнопку сама. Агент не публикует автономно.

## 2. Архитектура системы

Агент построен как цепочка трёх специализированных субагентов с общей памятью. Каждый делает одну задачу хорошо.

| Агент 1 — Сборщик новостей | Агент 2 — Редактор-голос | Агент 3 — Планировщик |
|---|---|---|
| Мониторит источники | Знает голос Светланы | Следит за балансом форматов |
| Фильтрует за 24ч | Генерирует 3 формата | Не повторяет темы |
| Отбирает топ-3 | Адаптирует стиль | Помнит что опубликовано |
| Проверяет уникальность | Добавляет продажу | Шлёт Светлане на выбор |

**Оркестратор (n8n / Make.com):** управляет цепочкой агентов, запускает по расписанию, передаёт данные между агентами, обращается к памяти.

## 3. Источники информации

### 3.1 MVP — Фаза 1 (запускаем сразу)

- RSS-ленты: Hugging Face Blog, OpenAI News, Google AI Blog, Anthropic Blog
- RSS-агрегаторы: TLDR AI, The Rundown AI, AINews.ai
- Google News RSS по запросам: «artificial intelligence», «нейросети», «AI tools»

### 3.2 Фаза 2 (после стабилизации MVP)

- Telegram-каналы конкурентов — список настраивается Светланой в Google Sheets
- X/Twitter — по хэштегам #AI #LLM #AITools (требует платного API $100+/мес — отдельное решение)

> ⚠️ Telethon для чтения TG-каналов несёт риск блокировки аккаунта Telegram. В Фазе 2 использовать только через отдельный технический аккаунт, не личный.

**Логика фильтрации новостей:**

1. Собрать все статьи за последние 24 часа
2. Отфильтровать не-ИИ контент
3. Проверить по архиву памяти — эта тема уже была в последние 7 дней?
4. Ранжировать по релевантности аудитории НейроСветы
5. Оставить топ-3 инфоповода
6. Если сигнала нет — пропустить день и уведомить Светлану

## 4. Система памяти агента

Без памяти агент каждый день начинает с нуля — не знает что публиковалось, повторяет темы, не учится. С памятью — он работает как настоящий помощник.

### 4.1 Хранилище памяти — Google Sheets (4 листа)

**Лист 1: `posts_archive` — архив публикаций**

| дата | тема | формат | источник | текст_поста | опубликовано |
|---|---|---|---|---|---|
| 17.06.2025 | GPT-5 релиз | новость | OpenAI RSS | текст... | да |

**Лист 2: `style_memory` — память стиля**

- Одобренные посты (✅) → пополняют банк примеров для промпта
- Отклонённые посты + причина отказа → агент учится что НЕ писать
- Любимые фразы и обороты Светланы — пополняется вручную
- Запрещённые слова и шаблоны (канцелярит, безликие ИИ-фразы)

**Лист 3: `format_balance` — баланс форматов**

- Счётчик: сколько раз за последние 7 дней был каждый формат
- Правило: не более 2 продающих постов подряд
- Правило: минимум 1 экспертный длинный пост в неделю

**Лист 4: `sources_config` — настройки источников**

- Список RSS-ссылок (Светлана добавляет/удаляет сама)
- Список TG-каналов конкурентов (Фаза 2)
- Курсы для продажи: название, ссылка, когда продвигать (по теме)
- Расписание запуска, часовой пояс

### 4.2 Как агент использует память

- Перед запуском читает архив последних 7 дней → не повторяет темы
- Читает `style_memory` → строит промпт с примерами голоса Светланы
- Читает `format_balance` → выбирает какой формат сегодня нужен
- После публикации записывает пост в архив
- Если Светлана нажала ✅ — сохраняет пост как новый пример стиля
- Если нажала ❌ с комментарием — записывает причину в `style_memory`

## 5. Обучение стилю автора

Агент должен писать КАК Светлана, а не как ChatGPT.

### 5.1 Два режима письма Светланы

| Режим 1: Эксперт думает вслух | Режим 2: Энергия и действие |
|---|---|
| Длинный пост, без хэштегов | Короткий продающий пост |
| Начало: личное наблюдение или вопрос | Начало: 💥 провокационный заголовок |
| Риторические вопросы по проблеме | Личная история → открытие |
| Честный взгляд без прикрас | Список с ✅ или цифрами |
| Короткие абзацы, воздух | ⚡️ Краткий вывод-формула |
| Практический вывод в конце | CTA: пиши мне @GolubkaSveta |
| Без хэштегов | Хэштеги: #нейросети #ИИ |
| 800–1500 символов | 400–800 символов |

### 5.2 Запрещённые паттерны

❌ «В условиях современной цифровой трансформации...» — канцелярит  
❌ «Искусственный интеллект открывает новые горизонты...» — безликий шаблон  
❌ Длинные абзацы без воздуха, монолитный текст  
❌ Пассивный залог: «было выявлено», «является важным»  
❌ Чрезмерные эмодзи в экспертном режиме  

### 5.3 Обязательные паттерны

✅ Начинать с конкретного наблюдения или личного опыта  
✅ Короткие абзацы — 1–3 предложения максимум  
✅ Разговорный язык: «просто хочу легко жить», «вот тут обычно тишина»  
✅ Практический вывод: что делать с этой информацией прямо сейчас  
✅ В продающем посте: CTA с @GolubkaSveta  

### 5.4 Механизм дообучения стилю

- Начальный промпт содержит 5–7 примеров лучших постов Светланы (Few-shot)
- После каждого одобренного поста (✅) — пост добавляется в `style_memory`
- Агент читает последние 10 одобренных постов перед каждой генерацией
- Раз в 2 недели: Светлана отмечает 3 лучших поста месяца → они получают высокий вес
- Опционально (Фаза 2): Светлана наговаривает голосовой комментарий к новости → агент пишет в этом тоне

## 6. Форматы постов

Каждое утро агент генерирует 3 поста — по одному формату. Светлана выбирает один для публикации.

### 🔥 Формат 1: Горячая новость + комментарий Светланы

- **Режим письма:** Эксперт думает вслух (Режим 1)
- **Структура:** Заголовок-крючок → Суть новости (2–3 предл.) → Почему это важно для аудитории → Авторский взгляд Светланы → Вопрос читателям
- **Длина:** 800–1200 символов
- **Хэштеги:** нет
- **Продажа:** нет (только экспертность)
- **Цель:** охваты, репосты, комментарии

### 🛠 Формат 2: Инструмент / Лайфхак дня

- **Режим письма:** смешанный — структура Режима 2, глубина Режима 1
- **Структура:** Проблема которую знает аудитория → Инструмент/решение → 3 конкретных шага → Результат → Призыв попробовать
- **Длина:** 600–1000 символов
- **Хэштеги:** 2–3 по теме инструмента
- **Продажа:** нет или очень мягко
- **Цель:** сохранения, подписки

### 📖 Формат 3: История / Кейс + мягкая продажа курса

- **Режим письма:** Режим 2 (энергия + личная история)
- **Структура:** Проблема-боль → Личная история Светланы → Как ИИ решил → Конкретный результат → Мягкий переход на курс → CTA
- **Длина:** 1000–1500 символов
- **Хэштеги:** #нейросети #ИИ + тематические
- **Продажа:** ДА — один из 2 курсов в зависимости от темы поста
- **Правило:** не более 2 формата-3 подряд, не чаще 3 раз в неделю
- **Цель:** прогрев, конверсия в продажу

### 6.1 Логика выбора курса для продажи

Агент знает 2 курса Светланы (данные в `sources_config`). При генерации Формата 3 выбирает курс исходя из темы поста — если пост про автоматизацию → курс об агентах, если про контент → курс по контенту с ИИ.

## 7. Интерфейс взаимодействия

Всё управление агентом — через личный Telegram-бот Светланы. Никаких сторонних сервисов, никаких веб-интерфейсов.

### 7.1 Утренний дайджест (8:00 МСК)

Агент присылает краткий дайджест топ-3 новостей:

```
📰 Доброе утро! Вот что горячего в ИИ сегодня:
1. [тема 1] — источник
2. [тема 2] — источник
3. [тема 3] — источник

Генерирую посты... ⏳
```

### 7.2 Посты на выбор

Через 1–2 минуты агент присылает 3 поста по очереди, каждый с кнопками:

| Кнопка | Действие |
|---|---|
| ✅ Опубликовать сейчас | Пост → канал + архив + обновляет `style_memory` |
| ⏰ Запланировать | Светлана выбирает время публикации |
| ✏️ Редактировать | Светлана вносит правки → агент применяет |
| 🔄 Перегенерировать | Агент пишет новый вариант этого же формата |
| 💬 Другой стиль | Агент переписывает пост в другом режиме (1↔2) |
| ❌ Пропустить | Пост отклонён + опционально причина → в `style_memory` |

### 7.3 Голосовой режим (Фаза 2)

- Светлана отправляет голосовое сообщение боту → агент транскрибирует → превращает в пост в её голосе
- Можно наговорить комментарий к конкретной новости: `/voice [ссылка на новость]`

### 7.4 Команды бота

| Команда | Описание |
|---|---|
| `/news` | Показать дайджест новостей прямо сейчас (вне расписания) |
| `/post [тема]` | Написать пост на конкретную тему |
| `/plan` | Показать контент-план на неделю с балансом форматов |
| `/style` | Показать текущий промпт стиля и примеры |
| `/stats` | Статистика: что опубликовано, какие форматы, архив |
| `/digest [url]` | Разобрать конкретную статью и написать пост по ней |

## 8. Технический стек

| Компонент | Решение |
|---|---|
| **Оркестратор** | n8n self-hosted (Railway.app или Hetzner VPS) — основной выбор |
| **Альтернатива** | Make.com — если n8n сложно, дороже, но без сервера |
| **ИИ-модель** | Claude Opus API (Anthropic) — для качества стиля |
| **Резервная модель** | GPT-4o (OpenAI) — fallback |
| **Источники новостей** | RSS Feed nodes в n8n + HTTP Request (GNews API) |
| **TG-парсер (Фаза 2)** | Telethon (Python) через отдельный технический аккаунт |
| **Память** | Google Sheets API — 4 листа (archive, style, balance, config) |
| **Telegram бот** | Telegram Bot API через n8n Telegram node |
| **Голос (Фаза 2)** | OpenAI Whisper API — транскрипция голосовых |
| **Планировщик** | n8n Schedule Trigger — 08:00 МСК ежедневно |
| **Хостинг** | Railway.app — $5–10/мес, простой деплой n8n |

> 💡 **Стартовая точка:** готовый n8n-шаблон «Daily AI News Digest» (n8n.io/workflows/9155) покрывает 70% задачи Агента 1. Скачать, адаптировать под НейроСвету — экономит 2–3 дня разработки.

## 9. План разработки — MVP → v2

### Фаза 1 — MVP (2–3 недели)

Цель: агент работает, посты генерируются, Светлана их видит и может опубликовать.

- [ ] Настройка n8n на Railway.app + базовый Telegram-бот
- [ ] RSS-сборщик новостей (5–7 источников) + фильтр за 24ч
- [ ] Google Sheets как память: 4 листа с базовой структурой
- [ ] Агент 2 (Редактор) — промпт с голосом Светланы (5 примеров постов)
- [ ] Генерация 3 форматов постов
- [ ] Telegram-интерфейс с кнопками (✅ / 🔄 / ❌)
- [ ] Архивирование опубликованных постов

### Фаза 2 — Улучшения (1–2 месяца после MVP)

Цель: агент учится на опыте, стиль становится точнее.

- [ ] Система дообучения стилю (одобренные посты → банк примеров)
- [ ] Баланс форматов — счётчик и правила
- [ ] Голосовой ввод через Whisper API
- [ ] Мониторинг TG-каналов конкурентов (Telethon, отдельный аккаунт)
- [ ] Команды `/stats`, `/plan`, `/digest`

### Фаза 3 — Рост (3–6 месяцев)

Цель: агент анализирует эффективность и сам улучшается.

- [ ] Интеграция аналитики канала (охваты, реакции) → агент знает какие посты заходят
- [ ] A/B тест заголовков
- [ ] Автоподбор картинки к посту (DALL-E / Midjourney API)
- [ ] X/Twitter API мониторинг

## 10. Риски и ограничения

| Риск | Уровень | Решение |
|---|---|---|
| Telethon — блокировка TG аккаунта | 🔴 Высокий | Только Фаза 2, отдельный технический аккаунт |
| Агент теряет голос Светланы | 🟡 Средний | Few-shot примеры в промпте + дообучение на одобренных |
| X/Twitter API стоимость $100+/мес | 🟡 Средний | Отложить, начать без Twitter |
| n8n падает — пост не генерируется | 🟡 Средний | Уведомление Светлане + ручной запуск `/news` |
| Стиль «размывается» со временем | 🟡 Средний | Регулярная ревизия `style_memory` раз в 2 недели |
| Два продающих поста подряд | 🟢 Низкий | Счётчик `format_balance` + жёсткое правило в промпте |
