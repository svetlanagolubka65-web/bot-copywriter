## 1. О проекте
Персональный ИИ-агент для Светланы Голубевой (@GolubkaSveta, t.me/nejroSvetlana) — эксперта по ИИ и владельца онлайн-школы. Каждое утро агент собирает новости об ИИ, генерирует 3 поста в авторском стиле Светланы и присылает на выбор в Telegram. Светлана всегда одобряет пост перед публикацией — агент не публикует автономно.

## 3. Tech Stack

| Слой | Инструмент | Назначение |
|---|---|---|
| Оркестратор | n8n self-hosted | Управление цепочкой агентов, расписание |
| Хостинг | Railway.app | Деплой n8n, ~$5-10/мес |
| ИИ-модель | Claude claude-opus-4-6 (Anthropic) | Генерация постов в стиле автора |
| Источники | RSS + GNews API | Сбор новостей об ИИ за 24ч |
| Память | Google Sheets API | 4 листа: архив, стиль, баланс, конфиг |
| Интерфейс | Telegram Bot API | Личный бот Светланы с кнопками |
| TG-парсер (v2) | Telethon (Python) | Мониторинг каналов конкурентов |

## 5. Ключевые решения

| # | Решение | Почему |
|---|---|---|
| 1 | **3 субагента вместо одного** | Каждый делает одну задачу хорошо; легко дебажить и менять отдельно |
| 2 | **Google Sheets как память** | Светлана видит и редактирует данные без кода; не требует БД |
| 3 | **Few-shot в промпте (не fine-tuning)** | Быстро, дёшево, стиль обновляется добавлением примеров в Sheets |
| 4 | **Человек в контуре обязателен** | Светлана жмёт ✅ сама — агент не публикует автономно, снижает риск |
| 5 | **Telethon в Фазе 2, не MVP** | Риск блокировки TG аккаунта высокий; RSS даёт 80% ценности без риска |

## 6. Тестирование

**Где:** локально в n8n (режим test workflow) перед деплоем на Railway.

**Как запускать:**
```bash
# Тест Agent1 — сборщик новостей
n8n execute --id=agent1_collector

# Тест Agent2 — генерация постов (нужен ANTHROPIC_API_KEY)
n8n execute --id=agent2_editor

# Полный прогон цепочки
n8n execute --id=agent3_planner
```

**Чек-лист перед деплоем:**
- [ ] Agent1 возвращает ≥1 новость за последние 24ч
- [ ] Agent2 генерирует ровно 3 поста (все форматы)
- [ ] Посты не дублируют темы из архива последних 7 дней
- [ ] Формат 3 содержит CTA с правильным курсом
- [ ] Telegram-бот доставляет сообщение с кнопками
- [ ] Google Sheets обновляется после публикации
- [ ] При нулевом сигнале — уведомление Светлане, не падение

## 9. Самообновление

Агент поддерживает актуальность своей памяти через следующий механизм:

| Триггер | Файл/лист | Действие |
|---|---|---|
| Светлана нажала ✅ | `sheets: style_memory` | Пост добавляется как новый few-shot пример |
| Светлана нажала ❌ + комментарий | `sheets: style_memory` | Причина отказа сохраняется → агент учится что НЕ писать |
| Раз в 2 недели | `prompts/style_voice.md` | Вручную обновить: взять топ-3 поста из style_memory → обновить few-shot |
| Новый курс / смена курса | `sheets: sources_config` | Светлана обновляет название + ссылку курса → агент подхватывает автоматически |
| Смена фазы (MVP→v2) | `CLAUDE.md` + `n8n/workflows/` | Добавить новые workflow, обновить структуру и архитектуру в этом файле |

> **Правило сессии:** в начале каждой новой сессии разработки — прочитать `CLAUDE.md` и `sheets: style_memory` (последние 10 строк). Это даёт полный контекст без лишних вопросов.
